# routing module ‚Äî build notes

**Rides:** GPT-5.6 (Sol/Terra/Luna), launched 2026-07-09 ‚Äî this is the core module the rest of the gateway wraps.

**What it does:**
- `ComplexityClassifier` ‚Äî cheap, zero-model-call heuristic (length + regex hard-signals + code-fence detection) that picks a **starting** tier. It only needs to be roughly right, because escalation catches the misses.
- `SmartRouteService` ‚Äî walks the tiers Luna ‚Üí Terra ‚Üí Sol, calling the model at each rung and checking the answer with a `Validator`, until one passes or Sol also fails. `route(prompt, validator)` uses the classifier's pick as the start; `routeFrom(prompt, validator, startTier)` forces a specific start tier instead (used by the gateway's governance layer ‚Äî see `gateway-NOTES.md`).

**The gotcha that matters most (see BUILD_NOTES.md):** cost is billed **per attempt at that attempt's own rate**, not at the final tier's rate. A Luna attempt that fails and escalates to Terra costs Luna-rate + Terra-rate, not just Terra-rate. Routing is not free ‚Äî a classifier that under-shoots on a hard task can cost *more* than going straight to Sol.

**Gotcha (resolved 2026-07-13):** that per-attempt cost was only tracked when every attempt in the loop actually got a response. If Luna succeeded (real cost incurred, answer rejected) and Terra's call itself then threw ‚Äî network error, upstream 5xx, timeout ‚Äî the bare exception propagated out of the loop and Luna's already-real cost vanished: never reached `RouteResult`, so never reached `SpendLedger` or `TelemetryService` either. Fixed with `PartialRouteException`, which carries a `RouteResult` snapshot of whatever was accumulated before the failing attempt (only earlier, *successful* attempts contribute cost ‚Äî the attempt that actually failed contributes zero, since no tokens came back). Callers that care about that partial cost catch it, record/book it, then rethrow so the failure still surfaces the same way it always did. See `gateway-NOTES.md` for the ledger side and `observability-NOTES.md` for the telemetry side.

**Endpoints:** `POST /route` (routing alone, no guardrails/governance wrapping ‚Äî see `GatewayController` / `POST /gateway/route` for the full pass).
**Tests:** `SmartRouteServiceTest` ‚Äî pass-at-start-tier, escalate-one-tier (with the per-attempt-cost assertion), climb-to-Sol-without-looping-forever, `routeFrom` overriding the classifier, and two `PartialRouteException` cases (failure after an earlier success preserves that cost; failure on the first attempt reports zero). `ComplexityClassifierTest` covers the tier-selection heuristic. All offline: `OpenAiChatModel` is mocked, `ChatResponse`/`Generation`/`AssistantMessage` are constructed as real objects (their accessors are `final` ‚Äî Mockito can't stub them).
